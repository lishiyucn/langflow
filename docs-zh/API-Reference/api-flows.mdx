使用 `/flows` 端点来创建、读取、更新和删除流程。

如果您想要使用 Langflow API 运行流程，请参阅[流程触发端点](/api-flows-run)。

## 创建流程

创建一个新流程。

```bash
curl -X POST \
  "$LANGFLOW_URL/api/v1/flows/" \
  -H "accept: application/json" \
  -H "Content-Type: application/json" \
  -H "x-api-key: $LANGFLOW_API_KEY" \
  -d '{
  "name": "string2",
  "description": "string",
  "icon": "string",
  "icon_bg_color": "#FF0000",
  "gradient": "string",
  "data": {},
  "is_component": false,
  "updated_at": "2024-12-30T15:48:01.519Z",
  "webhook": false,
  "endpoint_name": "string",
  "tags": [
    "string"
  ]
}'
```

## 读取流程

根据 ID 检索特定流程。

```bash
curl -X GET \
  "$LANGFLOW_URL/api/v1/flows/$FLOW_ID" \
  -H "accept: application/json" \
  -H "x-api-key: $LANGFLOW_API_KEY"
```

<summary>结果</summary>

```json
  "name": "Basic Prompting",
  "description": "Perform basic prompting with an OpenAI model.",
  "icon": "Braces",
  "icon_bg_color": null,
  "gradient": "2",
  "data": {
    "nodes": [
     ...
    ]
  }
}
```

</details>

## 读取多个流程

返回包含流程列表的 JSON 对象。

使用分页检索所有流程：

```bash
curl -X GET \
  "$LANGFLOW_URL/api/v1/flows/?remove_example_flows=false&components_only=false&get_all=true&header_flows=false&page=1&size=50" \
  -H "accept: application/json" \
  -H "x-api-key: $LANGFLOW_API_KEY"
```

要从特定项目检索流程，请使用 `project_id` 查询参数：

```bash
curl -X GET \
  "$LANGFLOW_URL/api/v1/flows/?remove_example_flows=true&components_only=false&get_all=false&project_id=$PROJECT_ID&header_flows=false&page=1&size=1" \
  -H "accept: application/json" \
  -H "x-api-key: $LANGFLOW_API_KEY"
```

```bash
  -H "x-api-key: $LANGFLOW_API_KEY"
```

## 更新流程

根据 ID 更新现有流程。

此示例将 `endpoint_name` 的值从随机 UUID 更改为 `my_new_endpoint_name`。

```bash
curl -X PATCH \
  "$LANGFLOW_URL/api/v1/flows/$FLOW_ID" \
  -H "accept: application/json" \
  -H "Content-Type: application/json" \
  -H "x-api-key: $LANGFLOW_API_KEY" \
  -d '{
  "name": "string",
  "description": "string",
  "data": {},
  "project_id": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
  "endpoint_name": "my_new_endpoint_name",
  "locked": true
}'
```

## 删除流程

根据 ID 删除特定流程。

```bash
curl -X DELETE \
  "$LANGFLOW_URL/api/v1/flows/$FLOW_ID" \
  -H "accept: application/json" \
  -H "x-api-key: $LANGFLOW_API_KEY"
```

<details>
<summary>结果</summary>

```json
{
  "message": "Flow deleted successfully"
}
```

</details>

## 导出流程

将指定流程导出为 ZIP 文件。

此端点下载包含请求主体中列出的每个流程 ID 的 [Langflow JSON 文件](/concepts-flows-import#langflow-json-file-contents)的 ZIP 文件。

```bash
curl -X POST \
  "$LANGFLOW_URL/api/v1/flows/download/" \
  -H "accept: application/json" \
  -H "Content-Type: application/json" \
  -H "x-api-key: $LANGFLOW_API_KEY" \
  -d '[
  "e1e40c77-0541-41a9-88ab-ddb3419398b5",
  "92f9a4c5-cfc8-4656-ae63-1f0881163c28"
]' \
  --output langflow-flows.zip
```

## 导入流程

通过上传 [Langflow 兼容的 JSON 文件](/concepts-flows-import#langflow-json-file-contents)导入流程。

要为流程指定目标项目，请包含查询参数 `project_id`。
在上传流程之前，目标 `project_id` 必须已存在。调用 [/api/v1/projects/](/api-projects#read-projects) 端点获取可用项目列表。

此示例将名为 `agent-with-astra-db-tool.json` 的本地文件上传到由 `PROJECT_ID` 变量指定的项目：

```bash
curl -X POST \
  "$LANGFLOW_URL/api/v1/flows/upload/?project_id=$PROJECT_ID" \
  -H "accept: application/json" \
  -H "Content-Type: multipart/form-data" \
  -H "x-api-key: $LANGFLOW_API_KEY" \
  -F "file=@agent-with-astra-db-tool.json;type=application/json"
```

<details>
<summary>结果</summary>

```json
[
  {
    "name": "agent-with-astra-db-tool",
    "description": "",
    "icon": null,
    "icon_bg_color": null,
    "gradient": null,
    "data": {}
  ...
  }
]
```

</details>
