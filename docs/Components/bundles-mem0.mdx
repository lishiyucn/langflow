[Bundle](/components-bundle-components) 包含支持与 Langflow 特定第三方集成的自定义组件。

本页面描述了 **Mem0** bundle 中可用的组件。

## Mem0 Chat Memory

**Mem0 Chat Memory** 组件使用 Mem0 内存存储检索和存储聊天消息。

### Mem0 Chat Memory 参数

**Mem0 Chat Memory** 组件的许多输入参数在可视化编辑器中默认隐藏。
您可以通过[组件标题菜单](/concepts-components#component-menus)中的 <Icon name="SlidersHorizontal" aria-hidden="true"/> **Controls** 切换参数。

| 名称 | 显示名称 | 信息 |
|------|--------------|------|
| mem0_config | Mem0 Configuration | 输入参数。用于初始化 Mem0 内存实例的配置字典。 |
| ingest_message | Message to Ingest | 输入参数。要接收到 Mem0 内存中的消息内容。 |
| existing_memory | Existing Memory Instance | 输入参数。可选的现有 Mem0 内存实例。 |
| user_id | User ID | 输入参数。与消息关联的用户标识符。 |
| search_query | Search Query | 输入参数。在 Mem0 中搜索相关内存的输入文本。 |
| mem0_api_key | Mem0 API Key | 输入参数。Mem0 平台的 API 密钥。留空以使用本地版本。 |
| metadata | Metadata | 输入参数。与接收消息关联的附加元数据。 |
| openai_api_key | OpenAI API Key | 输入参数。OpenAI 的 API 密钥。在使用 OpenAI 嵌入而没有提供配置时必需。 |

### Mem0 Chat Memory 输出

**Mem0 Chat Memory** 组件可以输出 **Mem0 Memory**（[`Memory`](/data-types#memory)）或 **Search Results**（[`Data`](/data-types#data)）。
您可以在组件输出端口附近选择输出类型。

使用 **Mem0 Chat Memory** 与 [**Message History** 组件](/components-helpers#message-history) 进行内存存储和检索操作。

使用 **Search Results** 输出根据搜索查询检索特定内存。
